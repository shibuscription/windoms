# Decisions

## 2026-02-15 Todayカレンダー範囲

- 判断: Todayには本格的な月間カレンダー（印刷対応含む）を入れない。
- 理由: Todayは日付ビュー（1日単位の確認と遷移）に責務を限定し、操作負荷と実装複雑度を抑えるため。
- 代替: 月間俯瞰・印刷要件は将来の別モジュールとして設計する。

## 2026-02-26 リンク集モジュール段階導入

- 判断: DEMOは実装速度優先でレベル2（`type` / `role` ベースの固定データ表示）を採用する。
- 本番想定: レベル3は Functions で OGP を取得し、キャッシュして配信する（クライアント直fetchはCORS/負荷観点で採用しない）。
- 互換方針: `ogTitle?` / `ogImageUrl?` / `faviconUrl?` / `host?` の optional fields 方式で段階導入し、データモデル互換を維持する。

## 2026-02-26 リンク集編集導線（DEMO）

- 判断: 独立編集画面は作らず、Linktree風に同一画面で編集モードを提供する。
- 配置: 追加アクションは右上共通アイコン群ではなく、画面タイトル横に配置する。
- 理由: 常駐ヘッダーに馴染ませすぎず、リンク集内の文脈で管理操作を完結させるため。

## 2026-03-03 購入依頼・立替モジュール方針（DEMO）

- 判断: 承認フローは採用しない。クラブ運用として「気づき→購入→精算」の抜け漏れ防止に特化する。
- 判断: 会計連携は片方向とする。会計レコード生成後は購入依頼・立替側で追跡しない。
- 判断: 立替は「ハンコ2つ」で完了とする（`PAID_BY_TREASURER` と `RECEIVED_BY_BUYER` が揃ったら `DONE`）。
